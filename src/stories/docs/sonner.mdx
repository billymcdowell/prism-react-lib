Displays temporary notifications.

<Canvas />

## Anatomy
1.  **Toaster**: The provider/viewport (usually at the root).
2.  **Toast**: The individual notification card.

## Behavior

### Interaction patterns
-   **Auto-dismiss**: Disappears after a duration (default 4s).
-   **Swipe**: Can swipe to dismiss.
-   **Action**: Can contain an action button.

<SectionHeader
  title="Usage Guidelines"
  description="Best practices for when and how to use the Sonner component."
/>

<Guidelines>
  <Dos>
    - **Feedback**: Use for success/error feedback after an action.
    - **Undo**: Provide an "Undo" action for destructive operations.
  </Dos>
  <Donts>
    - **Critical Info**: Hide critical information that the user must acknowledge (use Alert Dialog).
  </Donts>
</Guidelines>
