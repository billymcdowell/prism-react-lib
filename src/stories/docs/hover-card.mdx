## Behavior

### Interaction patterns
-   **Hover**: Opens after a short delay.
-   **Focus**: Opens on focus.

<SectionHeader
  title="Usage Guidelines"
  description="Best practices for when and how to use the Hover Card component."
/>

<Guidelines>
  <Dos>
    - **Previews**: Use for user profiles, repo details, or glossary terms.
    - **Metadata**: Show joined dates, follower counts, etc.
  </Dos>
  <Donts>
    - **Critical Actions**: Put buttons inside a hover card that are necessary for the user flow (hard to reach).
    - **Forms**: Put complex forms inside.
  </Donts>
</Guidelines>

## Accessibility

### Screen reader support
-   **Role**: `tooltip` (technically, but behaves like a dialog for content).
-   **Keyboard**: Users can Tab into the card if it remains open (Radix implementation ensures this).
