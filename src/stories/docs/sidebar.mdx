## Anatomy
![Anatomy Diagram Placeholder: Annotated diagram showing: 1. Provider, 2. Sidebar, 3. Header, 4. Content, 5. Footer, 6. Rail, 7. Trigger]

1.  **SidebarProvider**: Context provider for state (expanded/collapsed).
2.  **Sidebar (Root)**: The sidebar container.
3.  **SidebarHeader**: Top section (Logo, search).
4.  **SidebarContent**: Scrollable middle section (Menu).
5.  **SidebarFooter**: Bottom section (User profile).
6.  **SidebarRail**: Hoverable rail for resizing/expanding.
7.  **SidebarTrigger**: Toggle button.

## Behavior

### Interaction patterns
-   **Collapse**: Can collapse to icon-only mode.
-   **Mobile**: Turns into a Sheet (slide-in) on mobile.

## Implementation Notes

### Technical considerations
-   Uses CSS variables for width management.
-   Cookie-based state persistence (`sidebar_state`) to prevent hydration mismatch.
-   Composed of many sub-components (`SidebarMenu`, `SidebarGroup`, etc.).
