# Badge

<SectionHeader

Badges are small status descriptors for UI elements. They are typically used to communicate a status, a count, or a category.

## Variants
The Badge comes in several styles to communicate different statuses:

-   **Default**: Primary background color. Used for the main status or high-priority items.
    -   `variant="default"`
-   **Secondary**: Muted background color. Used for neutral or low-priority items.
    -   `variant="secondary"`
-   **Destructive**: Red background. Used for errors, critical warnings, or removal.
    -   `variant="destructive"`
-   **Outline**: Bordered transparent background. Used for subtle indicators.
    -   `variant="outline"`

## States
-   **Default**: The standard readable state.
-   **Hover**: Badges have a subtle hover effect (opacity change or background darken) if they are interactive or to add depth.

## Behavior

### Interaction patterns
-   **Static**: Badges are primarily informational and static.
-   **Link**: Can be used as a link if wrapped in an anchor tag (using `asChild` if supported, or just wrapping).

### Responsive behavior
-   **Wrapping**: Text should not wrap; badges should expand horizontally.
-   **Size**: Badges generally remain the same size across breakpoints.

    - Use for long sentences (keep text short).

## Accessibility

### Screen reader support
-   **Role**: Standard `span` or `div` unless interactive.
-   **Labeling**: Ensure the badge text is descriptive. If the badge is a status indicator, consider if additional context is needed for screen readers.

### WCAG compliance
-   **Contrast**: All variants meet AA contrast ratios.
    -   Default: `text-primary-foreground` on `bg-primary`.
    -   Destructive: `text-destructive-foreground` on `bg-destructive`.

## Design Specifications

### Typography
-   **Size**: `text-xs` (0.75rem).
-   **Weight**: `font-semibold` (600).
-   **Case**: Often Sentence case or Title Case, depending on usage.

### Spacing
-   **Padding**: `px-2.5` (horizontal), `py-0.5` (vertical).
-   **Border Radius**: `rounded-full` (Pill shape).

### Colors
-   **Default**: `bg-primary`.
-   **Secondary**: `bg-secondary`.
-   **Destructive**: `bg-destructive`.
-   **Outline**: `border-input` (or current border color).
