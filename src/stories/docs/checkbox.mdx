## Anatomy

1.  **Checkbox (Root)**: The clickable square box.
2.  **Indicator**: The check icon that appears when selected.
3.  **Label**: Text associated with the checkbox (external but related).

## States
-   **Unchecked**: Empty box.
-   **Checked**: Filled box with check icon.
-   **Indeterminate**: (Optional) Dash icon, used for nested lists.
-   **Disabled**: Reduced opacity, non-interactive.
-   **Focus**: Focus ring appears for keyboard navigation.

## Behavior

### Interaction patterns
-   **Click**: Toggles state.
-   **Space**: Toggles state when focused.

<SectionHeader
  title="Usage Guidelines"
  description="Best practices for when and how to use the Checkbox component."
/>

<Guidelines>
  <Dos>
    - **Binary Choices**: Use for on/off settings.
    - **Multiple Selection**: Use for selecting multiple items from a list.
    - **Labeling**: Always use with a label for click target size and accessibility.
  </Dos>
  <Donts>
    - Use for single selection from a list (use Radio Group).
    - Trigger immediate changes (unless it's a toggle/switch scenario, but Switch is preferred there).
  </Donts>
</Guidelines>

## Accessibility

### Screen reader support
-   **Role**: `checkbox`.
-   **State**: `aria-checked` ("true", "false", "mixed").
-   **Labeling**: Must be associated with a label via `id` and `for` (or wrapped).

### Keyboard navigation
-   **Tab**: Focuses the checkbox.
-   **Space**: Toggles selection.

## Design Specifications

### Spacing
-   **Size**: `size-4` (1rem).
-   **Radius**: `rounded-[4px]` (Small radius).

### Colors
-   **Checked**: `bg-primary` / `text-primary-foreground`.
-   **Border**: `border-primary` (checked) or `border-input` (unchecked).
